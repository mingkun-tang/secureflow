Security Insights

SecureFlow was built to explore how backend vulnerabilities actually happen in real systems, and how small mistakes can turn into serious security issues.

CSRF

State-changing actions must verify user intent, not just session identity.

IDOR

User-controlled identifiers should never decide access without server-side authorization checks.

Broken Access Control

Authentication is not enough. Systems must enforce what each user is actually allowed to do.

Attack Chains

Small flaws can combine into major impact, like account takeover through email change and password reset.

Featured Attack Chain

One of the main ideas behind SecureFlow is showing how small vulnerabilities can connect and lead to serious impact in a real backend system.

1

Exploit CSRF on Email Update

An attacker tricks a logged-in user into changing their email to an attacker-controlled address.

2

Trigger Password Reset

The attacker uses the insecure password reset flow that trusts only the email value.

3

Take Over the Account

The attacker gains control of the victim account through the reset process.

Final Impact: Full account takeover caused by chaining two backend security flaws together.
