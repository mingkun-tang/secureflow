Attack Demonstration

This page demonstrates how a normal user can escalate privileges due to insecure backend design.

Privilege Escalation Attack

  1. Login as a normal user (e.g. alice)
  2. Navigate to /change_role
  3. Enter role: boss
  4. Submit the form
  5. Navigate to /boss/financials
  6. Access restricted financial data

Secure Mode Behavior

In secure mode, only admin users can modify roles. Proper server-side authorization checks prevent privilege escalation.
